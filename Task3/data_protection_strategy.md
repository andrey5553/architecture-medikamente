| Данные                                        | Состояние                              | Чувствительность         | Профиль риска                            | Нормативная основа         |
| --------------------------------------------- | -------------------------------------- | ------------------------ | ---------------------------------------- | -------------------------- |
| ФИО, дата рождения, телефон, email            | В покое (файлы, БД), в движении (сеть) | Высокая (PII)            | Высокий (утечка → штраф, репутация)      | ФЗ-152                     |
| Медицинские данные (анализы, диагнозы, сканы) | В покое, в движении                    | Очень высокая (PHI)      | Критический (конфиденциальность, этика)  | ФЗ-152, ОСТ 42-01-96-86    |
| Адрес, место работы, хронические заболевания  | В покое                                | Высокая (избыточные PII) | Высокий (нарушение минимизации)          | ФЗ-152, GDPR-like          |
| Платежные данные (чеки, суммы)                | В покое, в движении                    | Высокая (финансы)        | Высокий (финансовая ответственность)     | 115-ФЗ                     |

| Данные сотрудников (зарплата, табель)         | В покое                                | Высокая                  | Средний                                  | ТК РФ, ФЗ-152              |
| Временные файлы (черновики, копии)            | В покое                                | Средняя → Высокая        | Средний (риск утечки через кэш)          | Внутренняя политика        |

# Инструменты защиты данных

## A. Шифрование данных при хранении (Encryption at Rest)

| Механизм                    | Применение                       | Решения                                                  |
| --------------------------- | -------------------------------- | -------------------------------------------------------- |
| Дисковое шифрование         | Файловый сервер, рабочие станции | BitLocker (Windows), LUKS (Linux)                        |
| Шифрование отдельных файлов | Excel, PDF, JPG с PII/PHI        | Adobe Acrobat (PDF), 7-Zip (AES-256)                     |
| Шифрование баз данных       | 1С в .1CD, будущая EMR           | Hashicorp Vault + AES                                    |
| Шифрование резервных копий  | Архивы, бэкапы                   | Veeam + шифрование, Acronis                              |
| Ключи шифрования            | Управление ключами               | Hashicorp Vault, Yandex Cloud KMS                        |

Рекомендация:
Внедрить централизованное управление ключами (Key Management System), чтобы избежать хранения ключей на тех же устройствах, где и данные.

## B. Шифрование данных при передаче (Encryption in Transit)

| Механизм                      | Применение                        | Решения                                   |
| ----------------------------- | --------------------------------- | ----------------------------------------- |
| TLS 1.2+ (HTTPS)              | Веб-интерфейсы, API, порталы      | Let's Encrypt, коммерческие сертификаты   |
| Шифрование OLE/COM-соединений | Интеграция ККМ и 1С               | Замена на HTTPS API с аутентификацией     |
| VPN / IPsec                   | Доступ к файловому серверу извне  | OpenVPN, WireGuard                        |
| SFTP / FTPS                   | Передача файлов (реестры, бэкапы) | WinSCP, FileZilla Pro                     |
| Email Encryption              | Отправка согласий, отчётов        | S/MIME, PGP, Microsoft Purview Encryption |

Рекомендация:
Отказаться от незащищённых протоколов (HTTP, FTP, Telnet). Все внутренние интеграции (в т.ч. ККМ ↔ 1С) должны использовать шифрованные каналы.

## C. Автоматизированный контроль за данными (Data Loss Prevention, DLP)

| Функция                               | Инструменты                     | Описание                                          |
| ------------------------------------- | ------------------------------- | ------------------------------------------------- |
| Автоматическое тегирование            | Microsoft Purview, Symantec DLP | Применение тегов #PII, #PHI на основе содержимого |
| Блокировка копирования                | Endpoint DLP                    | Запрет копирования #PHI в буфер, на USB           |
| Контроль email и облачных сервисов    | Microsoft 365 DLP               | Блокировка отправки PII/PHI на внешние email      |
| Мониторинг доступа к файлам           | SIEM + DLP                      | Алерты при попытке массового копирования          |
| Обнаружение утечек в реальном времени | Splunk, ELK + DLP-агенты        | Оповещение при попытке экспорта #PHI              |

Рекомендация:
Внедрить DLP-систему с политиками на основе тегов, чтобы автоматически блокировать рискованные действия.

# Итоговая таблица защиты данных

  1. Данные: Персональные данные (ФИО, дата рождения, телефон, email)
  В чём необходимость защиты: Обеспечение соответствия ФЗ-152 «О персональных данных».
  Инструменты защиты: BitLocker (шифрование диска), Hashicorp Vault (управление ключами), TLS 1.2+ (передача), Microsoft Purview (DLP, тегирование #PII)

  2. Данные: Медицинские данные (анализы, диагнозы, сканы медкарт)
  В чём необходимость защиты: Защита конфиденциальности пациентов (PHI), этическая и правовая ответственность. Утечка может повлечь уголовную ответственность и репутационные потери.
  Инструменты защиты: VeraCrypt (шифрование файлов), HTTPS (передача), DLP-система (блокировка копирования), аудит-логи (Wazuh/Splunk)

  3. Данные: Дополнительная информация (адрес, место работы, хронические заболевания)
  В чём необходимость защиты: Сбор избыточных данных требует повышенной защиты. Нарушение минимизации по ФЗ-152 при отсутствии согласия и защиты.
  Инструменты защиты: Шифрование в хранилище, тегирование #PII, обфускация при отображении, автоматическое удаление при отсутствии согласия

  4. Данные: Платежные данные (чеки, суммы, даты оплаты)
  В чём необходимость защиты: Защита финансовой информации, предотвращение мошенничества и расхождений в учёте. Соответствие требованиям безопасности транзакций.
  Инструменты защиты: Шифрование .1CD (1С), HTTPS для интеграции ККМ, SFTP для передачи отчётов, Veeam (зашифрованные бэкапы)

  5. Данные: Согласия на обработку персональных данных
  В чём необходимость защиты: Юридическое подтверждение легальности обработки. При утрате или отсутствии шифрования — риск признания обработки незаконной.
  Инструменты защиты: Шифрование файлов (AES-256), хранение в защищённой системе (Vault), аудит доступа, DLP (защита от копирования)

  6. Данные: Данные сотрудников (ФИО, зарплата, табель)
  В чём необходимость защиты: Защита персональных данных работников, соответствие ТК РФ и ФЗ-152. Риск утечек при неограниченном доступе.
  Инструменты защиты: BitLocker, RBAC (контроль доступа), шифрование в HR-системе, DLP-политики для бухгалтерии

  7. Данные: Временные файлы и копии (черновики, экспорт из 1С)
  В чём необходимость защиты: Высокий риск утечки через буфер обмена, USB, e-mail. Часто не учитываются в политике безопасности.
  Инструменты защиты: DLP (Forcepoint, Microsoft Purview), автоматическое удаление через 30 дней, запрет копирования #PHI/PII

  8. Данные: Резервные копии данных (бэкапы сервера, 1С, реестров)
  В чём необходимость защиты: Хранятся вне основной системы, могут быть утеряны или украдены. Должны быть защищены так же, как и основные данные.
  Инструменты защиты: Veeam / Acronis с шифрованием, хранение в РФ, контроль доступа к бэкап-серверу
